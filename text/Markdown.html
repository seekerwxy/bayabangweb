Markdown指南

1. 什么是 Markdown？

Markdown 是一种轻量级标记语言（Lightweight Markup Language），由 John Gruber 和 Aaron Swartz 于 2004 年共同创建。它允许人们使用易读易写的纯文本格式编写文档，然后通过工具转换成结构化的 HTML（或其他格式）。

其核心目标是：“让书写者专注于内容而非排版”。即使不经过转换，纯文本形态的 Markdown 也应当具备良好的可读性，就像一封普通的纯文本邮件一样自然。

2. 基础语法详解

以下语法基本在所有 Markdown 解析器中通用。

2.1 标题

使用 # 号，数量对应 HTML 的 <h1> 到 <h6>。

# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题

2.2 强调

删除线属于 GFM 扩展，但几乎被所有现代解析器支持。

语法效果
*斜体* 或 _斜体_斜体
**粗体** 或 __粗体__粗体
***粗斜体***粗斜体
~~删除线~~删除线

2.3 段落与换行

  • 直接连续的文字会自动合并成一段。
  • 段落间用空一行分隔。
  • 行末加两个空格（或使用 <br> 标签）可实现软换行（在最终输出中产生 <br>）。

2.4 列表

无序列表：使用 -、* 或 + 作为标记，后跟一个空格。

- 项目一
- 项目二
  - 嵌套项目（缩进两个空格）

有序列表：数字 + 英文句点 + 空格。

1. 第一步
2. 第二步
   1. 子步骤（缩进三个空格）

2.5 链接与图片

链接：[显示文本](https://example.com "可选的标题")

图片：在链接语法前加 !。
![替代文字](https://example.com/image.jpg "图片标题")

参考式链接（方便复用）：

[百度][baidu]
[baidu]: https://www.baidu.com "百度搜索"

自动链接：用尖括号包裹 URL 或邮箱，如 <https://example.com>、<someone@example.com>。

2.6 代码

行内代码：用反引号包裹。例如 请使用 `print("hello")` 函数。

代码块：用三个反引号（```）包裹，可指定语言以获得语法高亮（GFM）。

```python
def greet():
    print("Hello, World!")
```

也可通过缩进四个空格或一个制表符创建代码块，但无法指定语言。

2.7 引用

使用 > 符号，可嵌套。

> 这是一段引用。
> 
> > 嵌套引用。

2.8 水平分隔线

三个或更多的 ---、***、___，需独占一行。

---

2.9 转义字符

使用反斜线 \ 让特殊字符原样显示，如 \* 显示为 *。

3. 扩展语法（常见方言）

以下为 GFM（GitHub Flavored Markdown）及许多现代解析器支持的增强功能。

3.1 表格

| 左对齐 | 居中 | 右对齐 |
| :--- | :---: | ---: |
| 内容 | 内容 | 内容 |

渲染效果：

左对齐居中右对齐
内容内容内容

3.2 任务列表

- [x] 已完成任务
- [ ] 未完成任务

呈现样式（模拟）：

  • ☑ 已完成任务
  • ☐ 未完成任务

3.3 脚注（部分平台支持）

这里有一个脚注[^1]。

[^1]: 这是脚注的内容。

3.4 定义列表（部分扩展）

术语
: 定义内容

3.5 数学公式

需解析器支持（如 KaTeX、MathJax），通常用 $...$ 行内，$$...$$ 块级。

当 $a \ne 0$ 时，方程有解：
$$x = {-b \pm \sqrt{b^2-4ac} \over 2a}$$

3.6 Emoji 表情

GFM 支持短代码，如 :smile: 显示 😄，:+1: 显示 👍。也可直接复制粘贴 Unicode 表情。

3.7 目录生成

某些编辑器或解析器会根据标题自动生成 [TOC] 目录。

3.8 缩写、上标/下标

部分扩展提供，写法不一（如 Markdown Extra 的 ~下标~、^上标^，或 Pandoc 的 ~下标~、^上标^）。

4. 如何在留言板中使用 Markdown

编写建议：

  • 在“正文”输入框直接使用上述语法。
  • 配合彩色圆点生成的 [color=#rrggbb]文字[/color] 标签，可先写颜色标签，再在内部嵌套 Markdown，例如：
    [color=#ff0000]**红色的粗体祝福**[/color]。
  • 注意：颜色标签不是标准 Markdown，是自定义的，提交前会被 convertColorTags() 预处理成 HTML 后，再交给 Marked 解析，因此不要在颜色标签内使用换行或复杂语法。
  • 支持的行内样式：粗体、斜体、删除线、行内代码、链接等，均可与颜色叠加。
  • 代码块、表格、列表等块级元素同样支持，让你的留言更具表现力。
✅ 示例：[color=#2c6e2c]**_绿色的强调_**[/color] → 预处理后能得到带颜色的加粗文本。
